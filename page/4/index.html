工作经历总结

工作经历总结

Amazon Experience

分布式计算

wechat qrcode business

基于lambda serverless的微服务

  1. 为什么选择 Lambda？与传统服务器相比有什么优势？
    • 回答要点：按需扩展、无状态模型、低运维成本。
  2. 如何优化冷启动问题？
    • 回答要点：配置预热并发、减小打包体积、优化初始化逻辑。
  3. 高并发时如何处理流量限制？
    • 回答要点：API Gateway 限流、配置 Lambda 的并发限制、使用 SQS 进行队列化。
  4. 二维码生成中的缓存如何设计？
    • 回答要点：使用 S3 和 CDN 缓存固定二维码，减少重复生成。
  5. Lambda 的限制有哪些？如何应对？
    • 回答要点：运行时长限制（15 分钟）、冷启动、并发限制；应对方法包括任务分解、分批处理、配置预热实例等。

架构优化建议

如果采用 Lambda 和 Serverless 架构，可以针对二维码生成的需求进行以下优化：

高并发优化

  • 使用 API Gateway 限流，避免流量暴涨导致 Lambda 并发超额。
  • 配置 Provisioned Concurrency 保证冷启动时延降低。

静态二维码缓存：

  • 对于固定输入（如 URL、文本）生成的二维码，直接缓存结果（如存储在 S3 或 CDN 中），减少重复计算。
  • 配置 CloudFront 与 S3，将常用二维码放置到 CDN 节点，提高响应速度。

复杂任务分解：

  • 如果需要批量生成二维码，可以拆分任务，将单个二维码生成的请求分发给多个 Lambda 实例，结合 SQS 或 Step Functions 协调任务流程。

监控与日志：

  • 使用 CloudWatch 监控 Lambda 的运行时性能，及时发现冷启动问题或流量异常。
